# Дорожная карта НИР: МАС для полного цикла разработки ПО

## Цель

Разработать программу исследований для оценки мультиагентной системы,
выполняющей задачи полного цикла разработки ПО:

- анализ требований
- проектирование архитектуры
- планирование реализации
- разработка кода
- тестирование
- исправление дефектов
- подготовка итоговых артефактов

## Основные направления

### 1. Формализация полного цикла разработки

- определить этапы SDLC для МАС
- определить роли агентов
- определить правила handoff между агентами
- определить обязательную цепочку артефактов

### 2. Построение benchmark-задач

- bugfix
- небольшие feature-задачи
- refactoring
- test generation
- regression recovery

### 3. Оценка качества полного цикла

- качество итогового кода
- качество тестов
- полнота артефактов
- трассируемость решений между этапами
- согласованность архитектуры, плана и реализации

## Этапы работ

### Этап 1. Постановка исследования

- сформулировать исследовательские вопросы
- определить гипотезы
- выбрать состав ролей в МАС
- определить формат задач и артефактов

### Этап 2. Проектирование SDLC-benchmark

- описать единицу benchmark-задачи
- определить входные данные и ограничения
- определить ожидаемые выходные артефакты
- определить критерии завершения задачи

### Этап 3. Разработка инфраструктуры исполнения

- реализовать общий runner
- реализовать оркестрацию ролей
- реализовать хранение и учёт артефактов
- реализовать логирование A2A-взаимодействий

### Этап 4. Разработка средств оценки

- реализовать code checkers
- реализовать test checkers
- реализовать artifact checkers
- реализовать метрики стоимости, времени и качества

### Этап 5. Пилотный набор задач

- подготовить набор bugfix-задач
- подготовить набор feature-задач
- подготовить набор refactor-задач
- определить обязательные артефакты для архитектора и кодера

### Этап 6. Сравнение baseline-конфигураций

- single-agent baseline
- multi-agent baseline без учёта артефактов
- multi-agent baseline с учётом артефактов
- multi-agent baseline с recovery-механизмами

### Этап 7. Анализ результатов

- оценить качество полного цикла
- оценить вклад ролей и оркестрации
- оценить влияние артефактов на итоговый результат
- сформулировать рекомендации по построению МАС для SDLC

## Обязательные артефакты

- `requirements.md`
- `architecture.md`
- `implementation_plan.md`
- `delivery_report.md`
- `artifact_manifest.json`
- кодовые изменения
- тестовые изменения

## MVP

- 15-20 задач
- 3 типа сценариев: bugfix, feature, refactor
- 2-3 роли в МАС
- автоматическая оценка артефактов и тестов

## Критерии успеха

- воспроизводимый end-to-end запуск
- автоматическая проверка результатов
- сравнимость разных конфигураций МАС
- измеримая польза от оркестрации и handoff
- измеримая польза от учёта артефактов
